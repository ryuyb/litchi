# 通用审查规则

适用于所有语言的通用检查项。

---

## 代码质量

| 规则 | 严重程度 |
|------|----------|
| 函数单一职责，不超过 50 行 | 🟡 Medium |
| 命名清晰，遵循语言惯例 | 🟢 Low |
| 注释解释 "为什么" 而非 "是什么" | 🟢 Low |
| DRY 原则，避免重复代码 | 🟡 Medium |

---

## 测试

| 规则 | 严重程度 |
|------|----------|
| 关键业务逻辑有单元测试 | 🟠 High |
| API 端点有集成测试 | 🟡 Medium |
| 测试覆盖边界情况 | 🟡 Medium |

---

## Git 规范

| 规则 | 严重程度 |
|------|----------|
| Commit message 遵循 Conventional Commits | 🟢 Low |
| PR 粒度合理，一个 PR 做一件事 | 🟢 Low |
| 不提交调试代码和 .env 文件 | 🔴 Critical |

---

## 审查清单模板

审查完成时可用此清单汇总：

```markdown
## Review Checklist

### 正确性
- [ ] 错误处理完整
- [ ] 边界条件处理
- [ ] 并发安全 (Go) / Hooks 规则 (React)
- [ ] 类型安全

### 安全性
- [ ] 输入验证
- [ ] SQL 注入防护 (Go)
- [ ] XSS 防护 (React)
- [ ] 敏感数据处理

### 性能
- [ ] 无明显性能问题
- [ ] 数据库查询优化 (Go)
- [ ] 渲染优化 (React)

### 架构
- [ ] DDD 分层正确 (Go)
- [ ] 状态管理正确 (React)
- [ ] 依赖方向合理

### 测试
- [ ] 有对应的测试
- [ ] 测试覆盖关键路径
```
